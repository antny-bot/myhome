# 디지털주소정보플랫폼 오픈 API 활용 가이드 (도로명주소 검색 API)

행정안전부 도로명주소 안내 홈페이지([business.juso.go.kr](https://business.juso.go.kr))에서 제공하는 도로명주소 검색 오픈 API 활용 가이드입니다. 
도로명주소 DB를 별도로 구축하거나 관리할 필요 없이 다양한 시스템 환경에서 실시간으로 주소 검색 기능을 구현할 수 있습니다.

---

## 1. 서비스 소개

### 1.1. API 서비스 개요
* **다양한 시스템 환경 적용 가능**: 개발 언어나 플랫폼에 따른 제약이 없습니다.
* **DB 구축 및 현행화 부담 ZERO**: 실시간으로 도로명주소 도움센터의 데이터베이스에 접근하여 최신 주소를 제공합니다.
* **실시간 주소 검색**: 다양한 정렬 기능과 함께 주소를 실시간으로 조회합니다.
* **신청 즉시 이용 가능**: 오픈 API 이용 신청이 승인되면 즉시 승인키를 발급받아 사용할 수 있습니다.
* **1시스템 1승인키 부여**: 각 연계 시스템 또는 URL 기준으로 승인키가 부여됩니다.

### 1.2. 검색 기능 및 특징
* **변동된 주소정보 포함 (`hstryYn=Y`)**: 이전 지번 주소나 변동 전 도로명 주소로 검색하더라도 현재 시점의 도로명 주소를 자동으로 매핑하여 결과로 돌려줍니다.
* **행정동 명칭 지원**: 주민센터 등 행정동 명칭으로 검색하여도 이에 매핑되는 법정동 및 도로명 주소를 정상 조회합니다.
  > * **법정동**: 법률로 지정된 행정구역 (주소 및 공문서 등에서 사용)
  > * **행정동**: 행정운영 편의를 위해 설정한 행정구역 (관할 주민센터 기준)
* **다양한 정렬 기준 제공 (`firstSort`)**:
  * `none` (정확도순): 검색한 키워드와 가장 일치하는 주소정보 순으로 정렬
  * `road` (도로명 포함): 검색 키워드와 일치하는 도로명 주소를 우선 정렬
  * `location` (지번 포함): 검색 키워드와 일치하는 지번 주소를 우선 정렬

## 2. 프로그램 구현하기

### 2.1. API 기본 정보

| 항목 | 상세 정보 |
| :--- | :--- |
| **호출 방식** | `POST` 또는 `GET` |
| **요청 URL (일반)** | `https://business.juso.go.kr/addrlink/addrLinkApi.do` |
| **요청 URL (JSONP)** | `https://business.juso.go.kr/addrlink/addrLinkApiJsonp.do` |
| **출력 결과 형식** | XML 또는 JSON (JSONP 호출 시 JSONP 형식 반환) |

* *행정망 내 이용을 위한 URL은 도로명주소 도움센터(1588-0061)로 별도 문의 필요.*

### 2.3. 에러코드 정의 (errorCode)

| 에러코드 | 에러메시지 | 원인 및 해결방안 |
| :---: | :--- | :--- |
| **0** | 정상 | 정상적인 호출 및 처리 완료 |
| **-999** | 시스템에러 | 도로명주소 오픈 API 서버 오류 |
| **E0001** | 승인되지 않은 KEY 입니다. | 발급받은 승인키와 맞지 않거나 신청 방식(팝업/검색)이 다름 |
| **E0002** | 승인되지 않은 사이트 입니다. | 승인키 신청 시 작성한 URL/IP가 실제 호출하는 도메인과 불일치 |
| **E0003** | 정상적인 경로로 접속하시기 바랍니다. | 잘못된 헤더나 비정상적인 호출 요청 시 발생 |
| **E0005** | 검색어가 입력되지 않았습니다. | `keyword` 파라미터가 비어 있음 |
| **E0006** | 주소를 상세히 입력해 주시기 바랍니다. | 너무 포괄적이거나 잘못된 주소 형식 검색어 입력 |
| **E0008** | 검색어는 한글자 이상 입력되어야 합니다. | 검색어 길이가 너무 짧음 |
| **E0009** | 검색어는 문자와 숫자 같이 입력되어야 합니다. | 숫자만으로 구성된 검색어 입력 시 발생 (지번/도로명 이름이 필요) |
| **E0010** | 검색어가 너무 깁니다. | 한글 40자, 영문/숫자 80자 이하로 제한됨 |
| **E0011** | 검색어에 너무 긴 숫자가 포함되어 있습니다. | 숫자 10자 이하로 구성하여 검색 |
| **E0012** | 특수문자+숫자만으로는 검색이 불가능 합니다. | 무효한 특수문자 패턴 검색어 방지 |
| **E0013** | SQL 예약어 또는 특수문자( %,=,>,<,[,] )는 검색이 불가능 합니다. | 보안 규칙(Sql Injection)에 따른 차단 패턴 감지 |
| **E0014** | 개발승인키 기간이 만료되어 서비스를 이용하실 수 없습니다. | 임시 개발용 승인키 유효기간(최대 90일) 만료 |
| **E0015** | 검색 범위를 초과하였습니다. | 페이징 제한 범위를 넘어서는 대량 데이터 조회 시 발생 |

### 2.4. 자주 범하는 오류 및 보안 차단

#### 1) 자주 범하는 오류 사례
* **승인키 혼용 에러 (E0001)**: **팝업 API**용으로 신청한 승인키는 **검색 API** 호출 시 사용할 수 없으며, 그 반대도 마찬가지입니다. 구현 방식에 알맞게 승인키를 재발급받아야 합니다.
* **과도한 페이징 요청 차단**: 리스트 10개 기준으로 **900페이지**를 초과하는 범위를 페이지네이션할 경우 검색 범위 초과로 차단됩니다.
* **검색 키워드 인코딩**: 한글 검색을 원활하게 연동하기 위해 반드시 검색어(`keyword`)를 `UTF-8` 형식으로 인코딩하여 전송해야 합니다. (예: `URLEncoder.encode(keyword, "UTF-8")` 또는 `encodeURIComponent(keyword)`)

#### 2) API 보안 차단 규칙 (SQL Injection 패턴)
주소 검색어(`keyword`)에 아래와 같은 특정 문자열이 매칭될 경우 방화벽에 의해 해당 요청 IP 및 호출이 즉시 차단되거나 `E0013` 에러가 반환됩니다.
* **차단 규칙 패턴 예시**:
  * `'+or+'1'='1`
  * `' and 'f'='f`
  * `%27+and+`
  * `UNION+SELECT`
  * `%27+and+%27x%27%3D`
* **조치 방안**:
  API 호출 전에 클라이언트 또는 서버단에서 검색어 유효성을 체크하여, **SQL 예약어**(`or`, `select`, `insert`, `delete`, `update`, `create`, `drop`, `exec`, `union`, `fetch`, `declare`, `truncate` 등) 및 **특수문자**(`<`, `>`, `%`, `=`, `[` , `]` 등)를 필터링하거나 정제해야 합니다.

## 3. 이용 신청하기

### 3.1. 신청 절차
1. [주소기반산업지원서비스](https://business.juso.go.kr)에 접속 후 **주소정보 자료제공 > 주소정보 API 연계** 메뉴로 이동합니다.
2. 사용할 서비스 형태(검색 API, 팝업 API 등)를 선택하여 활용 신청합니다.
3. 신청 시 연계 목적, 개발사 및 연계 도메인 URL(또는 IP)을 작성해야 합니다.
4. 신청 완료 즉시 승인키를 획득할 수 있습니다. (개발용 키는 즉시 승인되나, 상용 서비스 연계 신청 시 일정 절차가 필요할 수 있습니다.)

### 3.2. 나의 신청 내역 조회 및 승인키 관리
* 상단의 **마이페이지 > API 인증키 관리** 메뉴에서 발급된 승인키의 상태(승인완료 여부)와 발급키 정보를 열람할 수 있습니다.
* 승인키는 발급 시 지정한 URL 또는 IP와 통신하는 요청에 대해서만 정상 작동하며, 사이트 변경 시 반드시 API 인증키 관리에서 수정 및 변경 신청해야 합니다.
