# 디지털주소정보플랫폼 오픈 API 활용 가이드 (도로명주소 검색 API)

행정안전부 도로명주소 안내 홈페이지([business.juso.go.kr](https://business.juso.go.kr)) 제공 도로명주소 검색 오픈 API 활용 가이드. 도로명주소 DB 별도 구축/관리 없이 다양한 시스템 환경에서 실시간 주소 검색 구현.

---

## 1. 서비스 소개

### 1.1. API 서비스 개요
* **다양한 시스템 환경 적용 가능**: 개발 언어/플랫폼 제약 없음.
* **DB 구축 및 현행화 부담 ZERO**: 도로명주소 도움센터 DB 실시간 접근으로 최신 주소 제공.
* **실시간 주소 검색**: 다양한 정렬 기능과 함께 주소 실시간 조회.
* **신청 즉시 이용 가능**: 오픈 API 이용 신청 승인 시 즉시 승인키 발급/사용 가능.
* **1시스템 1승인키 부여**: 각 연계 시스템/URL 기준 승인키 부여.

### 1.2. 검색 기능 및 특징
* **변동된 주소정보 포함 (`hstryYn=Y`)**: 이전 지번 주소 또는 변동 전 도로명 주소 검색 시 현재 도로명 주소 자동 매핑 반환.
* **행정동 명칭 지원**: 주민센터 등 행정동 명칭 검색 시 매핑 법정동/도로명 주소 조회 가능.
  > * **법정동**: 법률 지정 행정구역 (주소/공문서 사용)
  > * **행정동**: 행정운영 편의용 행정구역 (관할 주민센터 기준)
* **다양한 정렬 기준 제공 (`firstSort`)**:
  * `none` (정확도순): 검색 키워드와 일치 순 정렬.
  * `road` (도로명 포함): 일치 도로명 주소 우선 정렬.
  * `location` (지번 포함): 일치 지번 주소 우선 정렬.

## 2. 프로그램 구현하기

### 2.1. API 기본 정보

| 항목 | 상세 정보 |
| :--- | :--- |
| **호출 방식** | `POST` 또는 `GET` |
| **요청 URL (일반)** | `https://business.juso.go.kr/addrlink/addrLinkApi.do` |
| **요청 URL (JSONP)** | `https://business.juso.go.kr/addrlink/addrLinkApiJsonp.do` |
| **출력 결과 형식** | XML 또는 JSON (JSONP 호출 시 JSONP 형식 반환) |

* *행정망 전용 URL은 도로명주소 도움센터(1588-0061) 문의.*

### 2.3. 에러코드 정의 (errorCode)

| 에러코드 | 에러메시지 | 원인 및 해결방안 |
| :---: | :--- | :--- |
| **0** | 정상 | 정상 호출/처리 완료 |
| **-999** | 시스템에러 | 도로명주소 오픈 API 서버 오류 |
| **E0001** | 승인되지 않은 KEY 입니다. | 발급 승인키 불일치 또는 신청 방식 상이 |
| **E0002** | 승인되지 않은 사이트 입니다. | 신청 URL/IP와 실제 호출 도메인 불일치 |
| **E0003** | 정상적인 경로로 접속하시기 바랍니다. | 잘못된 헤더 또는 비정상 호출 요청 |
| **E0005** | 검색어가 입력되지 않았습니다. | `keyword` 파라미터 빈 값 |
| **E0006** | 주소를 상세히 입력해 주시기 바랍니다. | 포괄적이거나 잘못된 검색어 입력 |
| **E0008** | 검색어는 한글자 이상 입력되어야 합니다. | 검색어 길이 미달 |
| **E0009** | 검색어는 문자와 숫자 같이 입력되어야 합니다. | 숫자만으로 구성된 검색어 입력 |
| **E0010** | 검색어가 너무 깁니다. | 한글 40자, 영문/숫자 80자 초과 |
| **E0011** | 검색어에 너무 긴 숫자가 포함되어 있습니다. | 숫자 10자 초과 검색 |
| **E0012** | 특수문자+숫자만으로는 검색이 불가능 합니다. | 무효 특수문자 패턴 검색 방지 |
| **E0013** | SQL 예약어 또는 특수문자( %,=,>,<,[,] )는 검색이 불가능 합니다. | SQL Injection 차단 패턴 감지 |
| **E0014** | 개발승인키 기간이 만료되어 서비스를 이용하실 수 없습니다. | 임시 개발용 승인키 유효기간(90일) 만료 |
| **E0015** | 검색 범위를 초과하였습니다. | 페이징 제한 초과 대량 조회 |

### 2.4. 자주 범하는 오류 및 보안 차단

#### 1) 자주 범하는 오류 사례
* **승인키 혼용 에러 (E0001)**: **팝업 API**용 승인키는 **검색 API**에 사용 불가(역도 성립). 방식에 맞게 재발급 필요.
* **과도한 페이징 요청 차단**: 10개 기준 **900페이지** 초과 범위 페이지네이션 시 검색 범위 초과 차단.
* **검색 키워드 인코딩**: 한글 검색 위해 `keyword` `UTF-8` 인코딩 전송 필수. (예: `URLEncoder.encode(keyword, "UTF-8")`, `encodeURIComponent(keyword)`)

#### 2) API 보안 차단 규칙 (SQL Injection 패턴)
주소 검색어(`keyword`) 특정 SQL Injection 패턴 매칭 시 IP 차단 또는 `E0013` 에러 반환.
* **차단 규칙 패턴 예시**:
  * `'+or+'1'='1`
  * `' and 'f'='f`
  * `%27+and+`
  * `UNION+SELECT`
  * `%27+and+%27x%27%3D`
* **조치 방안**:
  호출 전 클라이언트/서버단 검색어 유효성 검사 수행. **SQL 예약어**(`or`, `select`, `insert`, `delete`, `update`, `create`, `drop`, `exec`, `union`, `fetch`, `declare`, `truncate` 등) 및 **특수문자**(`<`, `>`, `%`, `=`, `[`, `]` 등) 필터링/제거.

## 3. 이용 신청하기

### 3.1. 신청 절차
1. [주소기반산업지원서비스](https://business.juso.go.kr) 접속 후 **주소정보 자료제공 > 주소정보 API 연계** 이동.
2. 서비스 형태(검색 API, 팝업 API 등) 선택 후 신청.
3. 연계 목적, 개발사, 도메인 URL/IP 작성.
4. 신청 완료 즉시 승인키 획득(개발용 즉시 승인, 상용은 별도 절차 가능).

### 3.2. 나의 신청 내역 조회 및 승인키 관리
* **마이페이지 > API 인증키 관리** 메뉴에서 발급 승인키 상태/정보 열람 가능.
* 발급 시 지정한 URL/IP 요청만 작동. 변경 시 API 인증키 관리에서 수정 신청 필수.
