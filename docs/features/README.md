# 기능별 상세 문서 (Features Index)

> 이 폴더는 **구현된 기능별 상세 문서**를 모읍니다.
> 에이전트는 이 인덱스에서 **작업과 관련된 문서 한 개만** 골라 읽으세요. 전체를 읽지 마세요.

## 사용법 (토큰 절약 규칙)

1. 먼저 이 표에서 작업 대상 기능을 찾는다.
2. 해당 행의 "상세 문서" **하나만** 연다.
3. 표의 "관련 파일"은 그 문서를 읽기 전 빠른 위치 확인용이다.

## 기능 목록

| 기능 | 한 줄 설명 | 상세 문서 | 관련 파일(요약) |
|------|-----------|-----------|----------------|
| 지역 검색 / 주소 자동완성 | 지역명 입력 시 카카오 주소 검색으로 하위 후보를 좁혀 선택. 키 없으면 MCP 단일 검색 폴백 | [`region-search.md`](./region-search.md) | `server/addressSearch.ts`, `server/routes.ts`, `src/App.tsx` |

## 새 기능 문서를 추가할 때

- 위 표에 **행 한 줄**을 추가하고(한 줄 설명 + 문서 링크 + 관련 파일),
- `docs/features/<기능명>.md` 에 상세 문서를 작성한다.
- 상세 문서는 **자체 완결적**으로 작성한다(그 문서만 읽어도 작업 가능하도록).
- `docs/OVERVIEW.md`의 "작업 유형별 필독 파일" 표에도 필요한 경우 행을 추가한다.
